# 基于当前后端数据的手搓神经网络计划

## 1. 当前后端生成的数据规模

当前接口 `/grayImage/generateGrayDatas` 虽然接收 `SyntheticDigitGenerateRequest`，但 `GrayImageService.generateDefaultGrayImages` 会重新构造请求并固定参数：

- `count`: 1 到 30
- `width`: 128
- `height`: 128
- `digitsPerImage`: 4
- `noiseLevel`: 0.2
- 图像格式: PNG
- 图像类型: `BufferedImage.TYPE_BYTE_GRAY`
- 标签: `imgValue`，例如 `"0427"`，表示 4 位数字

所以当前新生成样本的源数据尺寸是：

- 单张图像逻辑尺寸: `1 × 128 × 128`
- 像素数: `128 × 128 = 16384`
- 原始灰度数据: `uint8`，每张约 `16384 B = 16 KiB`
- 神经网络输入: 建议转成 `float32`，每张约 `16384 × 4 = 65536 B = 64 KiB`
- 一次接口最多生成 30 张，转成 `float32` 后输入约 `30 × 64 KiB = 1.875 MiB`

当前磁盘目录 `D:/YueShaoPu/Web3D/png/datasets/synthetic-digits` 里已有混合历史数据：

| 尺寸 | 数量 | PNG 文件大小 |
| --- | ---: | --- |
| `128 × 128` | 30 | 约 `9670 ~ 10857 B`，平均约 `10.1 KiB` |
| `160 × 64` | 34 | 约 `6693 ~ 7162 B`，平均约 `6.7 KiB` |

训练时以当前代码新生成的 `128 × 128` 为准。历史 `160 × 64` 数据建议先跳过，或者统一 resize 到 `128 × 128` 后再混入。

## 2. 任务定义

不要把 4 位数字字符串当成 `0000 ~ 9999` 的 10000 分类问题。当前生成器每张图固定有 4 个位置，每个位置是 0 到 9，因此更合理的输出是：

```text
input:  1 × 128 × 128
output: 4 × 10 logits
loss:   4 个位置分别做 softmax cross entropy，再取平均
```

例如标签 `"0427"` 转成：

```text
target[0] = 0
target[1] = 4
target[2] = 2
target[3] = 7
```

这样输出层只有 40 个 logit，训练更稳定，手搓反向传播也更清楚。

## 3. 推荐主方案：整图 4 头 CNN

这个方案直接吃完整 `128 × 128` 图片，不需要先裁剪。参数量约 `145K`，对 C++ 手搓前向和反向都比较合适。

| 层 | 配置 | 输出尺寸 | 参数量 |
| --- | --- | --- | ---: |
| Input | 灰度图 | `1 × 128 × 128` | 0 |
| Conv2D | `3×3, stride=1, pad=1, out=8` | `8 × 128 × 128` | `8×1×3×3+8 = 80` |
| ReLU | - | `8 × 128 × 128` | 0 |
| MaxPool | `2×2, stride=2` | `8 × 64 × 64` | 0 |
| Conv2D | `3×3, stride=1, pad=1, out=16` | `16 × 64 × 64` | `16×8×3×3+16 = 1168` |
| ReLU | - | `16 × 64 × 64` | 0 |
| MaxPool | `2×2, stride=2` | `16 × 32 × 32` | 0 |
| Conv2D | `3×3, stride=1, pad=1, out=24` | `24 × 32 × 32` | `24×16×3×3+24 = 3480` |
| ReLU | - | `24 × 32 × 32` | 0 |
| MaxPool | `2×2, stride=2` | `24 × 16 × 16` | 0 |
| Conv2D | `3×3, stride=1, pad=1, out=32` | `32 × 16 × 16` | `32×24×3×3+32 = 6944` |
| ReLU | - | `32 × 16 × 16` | 0 |
| MaxPool | `2×2, stride=2` | `32 × 8 × 8` | 0 |
| Flatten | - | `2048` | 0 |
| Linear | `2048 → 64` | `64` | `2048×64+64 = 131136` |
| ReLU | - | `64` | 0 |
| Linear | `64 → 40` | `4 × 10` | `64×40+40 = 2600` |
| SoftmaxCE | 每个数字位置独立 softmax | `4` 个 loss | 0 |

总参数量：

```text
80 + 1168 + 3480 + 6944 + 131136 + 2600 = 145408
```

`float32` 权重本体约 `145408 × 4 = 568 KiB`。如果同时保存梯度和动量，参数相关内存也只是 1 到 2 MB 量级。单样本反向传播保留中间特征图时，激活内存大约 1 MB 级别，可控。

## 4. 为什么不继续用旧的 256×256 方案

旧计划按 `256 × 256` 估算，输入维度是 65536；当前后端实际新生成的是 `128 × 128`，输入维度只有 16384。

如果继续用旧方案的 `256×256 → 32×32×32 → Linear(32768, 128)`，会把全连接层做得过胖。当前 `128×128` 下更适合多池化一次，把特征压到 `32×8×8 = 2048` 后再接小全连接层。

旧方案约 4.2M 参数；当前推荐方案约 145K 参数，参数量缩小约 29 倍，手搓训练、调试梯度和保存权重都更现实。

## 5. 数据读取和预处理

建议 C++ 侧先用 OpenCV 读取 PNG，但神经网络计算部分自己写。

```cpp
cv::Mat img = cv::imread(path, cv::IMREAD_GRAYSCALE);
CV_Assert(img.rows == 128 && img.cols == 128 && img.type() == CV_8UC1);

std::vector<float> x(128 * 128);
for (int y = 0; y < 128; ++y) {
    for (int x0 = 0; x0 < 128; ++x0) {
        x[y * 128 + x0] = img.at<unsigned char>(y, x0) / 255.0f;
    }
}
```

标签从 `imgValue` 字符串解析：

```cpp
std::array<int, 4> y = {
    label[0] - '0',
    label[1] - '0',
    label[2] - '0',
    label[3] - '0'
};
```

内存布局建议统一用 `CHW`：

```text
index = (c * H + h) * W + w
```

原因是卷积、池化、Flatten 的形状更容易手动检查。当前输入只有 1 个通道，后续卷积输出多通道后也方便按通道调试。

## 6. 训练配置建议

先不要上 BatchNorm、Dropout、残差连接。这些东西会增加很多状态和反向传播分支，第一版不值得。

推荐第一版训练配置：

- 初始化: ReLU 层使用 He 初始化，范围可用 `sqrt(2 / fan_in)`
- 优化器: 先写 SGD + Momentum
- 学习率: `0.01` 起步，如果 loss 抖动就降到 `0.003`
- Momentum: `0.9`
- Batch size: 先用 `1` 做梯度检查，跑通后用 `8` 或 `16`
- Loss: 4 个位置的交叉熵平均
- Accuracy: 同时记录单字符准确率和整串准确率

整串准确率更严格：

```text
label = 0427
pred  = 0427  => 正确
pred  = 0421  => 整串错误，但单字符 3/4 正确
```

## 7. 手搓实现顺序

按这个顺序写，调试成本最低：

1. `Tensor` 结构：保存 `std::vector<float> data` 和 `C/H/W` 或一维长度。
2. 数据加载：读取 PNG，归一化到 `[0, 1]`，解析 4 位标签。
3. `Linear` 前向和反向。
4. `ReLU` 前向和反向。
5. `SoftmaxCrossEntropy4`：输入 40 个 logit，输出 loss 和 `dlogits[40]`。
6. `MaxPool2D` 前向和反向：前向时保存最大值位置。
7. `Conv2D` 前向。
8. `Conv2D` 反向：分别算 `dInput`、`dWeight`、`dBias`。
9. 串起完整网络，先单样本过拟合，再扩展到小批量。

必须做数值梯度检查，尤其是 `Conv2D`：

```text
numeric_grad = (loss(w + eps) - loss(w - eps)) / (2 * eps)
eps = 1e-3
相对误差 < 1e-2 可接受
```

卷积层因为浮点误差和循环累加顺序，手写时不必强求 `1e-5`。

## 8. 可选更简单方案：按 4 个数字格裁剪

当前生成器把 4 个数字画在 4 个等宽 cell 里，因此也可以把 `128 × 128` 切成 4 张 `128 × 32` 小图，每张做 10 分类，并共享同一个小 CNN。

这个方案参数约 `104K`，更接近“单数字识别”，但要写 4 次前向并累加共享权重梯度。

单个裁剪网络：

| 层 | 输出尺寸 | 参数量 |
| --- | --- | ---: |
| Input | `1 × 128 × 32` | 0 |
| Conv `3×3, out=8` + ReLU + Pool | `8 × 64 × 16` | 80 |
| Conv `3×3, out=16` + ReLU + Pool | `16 × 32 × 8` | 1168 |
| Conv `3×3, out=24` + ReLU + Pool | `24 × 16 × 4` | 3480 |
| Flatten | `1536` | 0 |
| Linear `1536 → 64` + ReLU | `64` | 98368 |
| Linear `64 → 10` | `10` | 650 |

主线仍建议先做“整图 4 头 CNN”，因为代码路径更直接，后续如果发现定位困难，再切换到裁剪共享 CNN。

## 9. 当前阶段目标

第一阶段不要追求复杂模型，目标是把完整训练链路跑通：

- 能读取后端生成的 `128 × 128` PNG。
- 能把 `imgValue` 转成 4 个数字标签。
- 能前向得到 `4 × 10` logits。
- 能反向更新卷积层和全连接层。
- 能在 30 到 300 张样本上明显过拟合。
- 再生成更多批次做训练/验证划分。

只要这条链路打通，再考虑更深网络、数据增强、学习率衰减和模型保存格式。
